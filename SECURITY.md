# Política de seguridad

## Estado del proyecto

PETTZI se publica actualmente como repositorio público de referencia técnica. No representa un SaaS operado activamente por el autor.

La infraestructura y los flujos documentados existen con fines de referencia, aprendizaje o self-hosting bajo responsabilidad propia.

## Reporte de vulnerabilidades

Si encuentras una vulnerabilidad de seguridad en el código o en la documentación:

- no publiques detalles sensibles en un issue público
- usa el canal privado de GitHub Security Advisories si está disponible
- si ese canal no está habilitado, contacta al autor mediante el correo público asociado a su perfil de GitHub

## Qué incluir en el reporte

Incluye, si es posible:

- descripción del problema
- impacto estimado
- pasos para reproducir
- archivos o componentes afectados
- posible mitigación

## Alcance

Se agradecen reportes relacionados con:

- autenticación y autorización
- validación de inputs
- exposición de datos sensibles
- errores de configuración inseguros en infraestructura documentada
- manejo inseguro de credenciales, tokens o URLs firmadas

## Expectativas

Haré el mejor esfuerzo por revisar reportes razonables, pero no puedo prometer SLA, soporte continuo ni releases en tiempos determinados.
