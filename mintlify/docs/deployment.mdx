# Deployment

> Public repo note: deployment guidance is preserved for reference and self-hosting. The GitHub Actions publishing workflow is intentionally disabled because this project is no longer operated as a commercial SaaS.

## Prerequisites

- AWS credentials / SSO.
- CDK bootstrapped in target account/region.
- `.env` with `STAGE`, `AWS_PROFILE`, `AWS_REGION`, `SES_FROM_EMAIL`, and either:
  - `API_DOMAIN_NAME` (full domain), or
  - `API_PREFIX` + `API_HOSTED_ZONE_NAME` (prefix + hosted zone). Add `API_HOSTED_ZONE_ID` to avoid Route53 lookups.

## Build

```bash
npx nx run cdk:build:production
```

## Deploy stacks (typical order)

1. `PettziApplicationStack` (AppRegistry app)
2. `PettziLayersStack`
3. `PettziCoreInfraStack`
4. `PettziAuthStack`
5. API stacks: `PettziAuthApiStack`, `PettziPetsApiStack`, `PettziOwnersApiStack`, `PettziEventsApiStack`, `PettziRemindersApiStack`, `PettziUploadsApiStack`, `PettziCatalogsApiStack`
6. `PettziSesTemplatesStack`
7. `PettziApiDomainStack`
8. `PettziAppRegistryAssociationsStack` (associations to AppRegistry)

Command example:

```bash
npx nx run cdk:deploy -- PettziApplicationStack
npx nx run cdk:deploy -- PettziLayersStack PettziCoreInfraStack PettziAuthStack
npx nx run cdk:deploy -- PettziPetsApiStack PettziOwnersApiStack
npx nx run cdk:deploy -- PettziApiDomainStack
npx nx run cdk:deploy -- PettziAppRegistryAssociationsStack
```

## Env vars per stack (high level)

- Core: none beyond STAGE.
- Auth API: `SES_FROM_EMAIL`, template names if overriding defaults.
- Reminders: `REMINDERS_EMAIL_FROM`, `SES_REMINDER_TEMPLATE_NAME`.
- Uploads: `PETTZI_DOCS_BUCKET_NAME` (set by stack).
- Domain: either `API_DOMAIN_NAME` or `API_PREFIX` + `API_HOSTED_ZONE_NAME` (`API_HOSTED_ZONE_ID` optional).
